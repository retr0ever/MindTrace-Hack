How to Use MindTrace

MindTrace is an automated EEG signal processing platform. It allows researchers to upload raw brainwave data and receive cleaned, research-ready datasets in seconds.

Getting Started

  1. Prepare your data: Ensure your EEG recording is in .csv or .npy format.
  2. Upload: Drag and drop your file onto the Analysis page.
  3. Wait for processing: The system will automatically apply filters and remove noise.
  4. Download results: You can download the cleaned dataset, a full scientific report, or listen to the AI audio summary.
Note: No manual configuration is required. The pipeline automatically detects sampling rates and channel configurations.

Use Cases

MindTrace is designed for researchers and clinicians working with EEG data across various domains:

Academic Research

Quickly clean and prepare EEG datasets for publication-ready analysis. Ideal for cognitive neuroscience studies, sleep research, and brain-computer interface development.

Clinical Diagnostics

Pre-process patient EEG recordings to identify neural patterns obscured by artefacts. Useful for epilepsy monitoring, sleep disorder assessment, and cognitive impairment screening.

Education & Training

Teach students about EEG signal processing with an intuitive interface. The visual waveform comparison and detailed reports help explain filtering and artefact removal concepts.

Brain-Computer Interfaces (BCI)

Prepare clean training data for BCI systems by removing eye blinks, muscle activity, and electrical interference from raw recordings.

Longitudinal Studies

Process large batches of EEG recordings consistently using the same pipeline parameters, ensuring reproducibility across sessions and participants.

Algorithms Used

Signal Pre-processing

We use a standard neuroscience filter stack to prepare the data:

  • Bandpass Filter (1–40 Hz): A Butterworth filter designed to remove very slow drifts (like movement artifacts) and high-frequency noise (e.g., muscle activity, external electrical interference).
  • Notch Filter (50/60 Hz): Applied to precisely eliminate power line interference (mains hum), a common electrical artifact in many labs.

Artefact Removal: Independent Component Analysis (ICA)

ICA is a powerful technique used to separate mixed signals into their underlying independent sources. MindTrace employs FastICA to identify and effectively remove common physiological artifacts, such as:

  • Eye Blinks (EOG): These are usually large, transient deflections in frontal channels.
  • Muscle Activity (EMG): Appears as high-frequency noise across channels.

Analysis & Metrics

The system calculates several key metrics to quantify signal quality improvement and characterize brain activity:

SNR Improvement

Signal-to-Noise Ratio (SNR) improvement is calculated by comparing the signal variance before and after cleaning. A higher dB value indicates a more effective noise reduction.

Dominant Frequency Bands

Power Spectral Density (PSD) analysis using Welch's method is performed to determine the relative power in different EEG frequency bands, helping to infer brain states:

  • Delta (0.5-4 Hz): Associated with deep sleep states.
  • Theta (4-8 Hz): Linked to drowsiness, meditation, and memory processing.
  • Alpha (8-13 Hz): Dominant during relaxed wakefulness, often with eyes closed.
  • Beta (13-30 Hz): Reflects active thinking, focus, and alertness.
  • Gamma (>30 Hz): Implicated in high-level cognitive functions and sensory binding.

AI Integration

MindTrace utilizes AI beyond just signal processing, primarily for user interaction and clear communication of results:

  • SpoonOS Agent Integration: (If configured) Allows natural language interaction to control specific aspects of the processing pipeline, such as adjusting parameters or undoing certain cleaning steps.
  • Automated Report Generation: An AI-powered system structures the analysis results into a comprehensive, human-readable scientific report.
  • ElevenLabs TTS Briefing: (If configured) Converts the key findings from the analysis into a natural-sounding audio summary, providing an accessible overview of the results.

Current Limitations

While MindTrace provides robust EEG processing capabilities, there are some current limitations to be aware of:

  • ICA Requirements: ICA requires at least two channels to meaningfully separate sources. Single-channel data falls back to filtering only.
  • Sampling Rate: The pipeline assumes a 256 Hz sampling rate by default. Data with significantly different rates may require configuration adjustments.
  • Artefact Detection: Automatic artefact component detection uses threshold-based heuristics. Complex or unusual artefacts may require manual review.
  • File Size: Very large recordings (>1GB) may cause memory issues on systems with limited RAM.
  • File Formats: Currently supports .csv and .npy formats only. EDF/BDF support is planned.
  • Browser Support: The web interface is optimised for modern browsers (Chrome, Firefox, Safari).

Future Plans

We are actively developing new features to enhance MindTrace:

Near-term Roadmap

  • Multi-format Support: Add support for EDF, BDF, and MNE-Python raw formats commonly used in clinical and research settings.
  • Batch Processing: Enable processing of multiple files simultaneously with progress tracking and consolidated reporting.
  • Custom Pipeline Builder: Allow users to configure their own processing pipeline by selecting and ordering individual steps.

Long-term Vision

  • Real-time Processing: Stream EEG data directly from acquisition devices for live artefact removal and monitoring.
  • ML-based Artefact Detection: Implement machine learning-based artefact classification for more accurate automatic component rejection.
  • Collaborative Features: Add user accounts, shared workspaces, and annotation tools for team-based research.
  • REST API: Provide programmatic access to the cleaning pipeline for integration with other research tools.
  • Extended Analytics: Add event-related potential (ERP) analysis, connectivity metrics, and source localisation capabilities.
